# DTS2026062420586: CDN 404 临时故障导致转存任务终止

## 问题描述
百度网盘 CDN 临时性故障返回 HTTP 404 nginx HTML 页面，而非正常的 JSON API 响应。现有代码将此视为不可重试错误，导致整个转存任务终止。

## 问题时间
- 发现时间：2026-06-24
- 修复时间：2026-06-24

## 影响范围
- 功能：文件转存
- 严重程度：高（任务中断）

## 根因分析
百度 CDN 有时返回 HTTP 404 nginx HTML 页面，这是临时性故障。`get_share_children()` 函数中：
1. HTTP 404 不在 `_SERVER_ERROR_CODES` 列表中
2. `safe_json_parse` 返回错误后，代码直接返回错误不重试

## 修复方案
1. 新增 CDN 404 重试配置：
   - `_CDN_404_MAX = 3`：最大重试次数
   - `_CDN_404_BASE_DELAY = 5`：基础延迟秒数
   - `_CDN_404_DELAY_STEP = 5`：延迟递增秒数

2. 在 `get_share_children()` 中添加重试逻辑：
   - 检测 `resp.status_code == 404`
   - 重建 httpx 连接清除可能的坏连接
   - 延迟递增重试（5→10→15秒）
   - 超过重试次数才返回错误

## 修复文件
- `baidu_api.py`：添加 CDN 404 重试逻辑

## 测试验证
- ✅ HTTP 404 重试后成功
- ✅ HTTP 404 重试次数用尽
- ✅ 普通 API 错误不重试
- ✅ HTTP 500 服务器错误重试
- ✅ safe_json_parse 正常/异常响应

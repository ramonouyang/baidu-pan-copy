# DTS2026062568561 — 文件列表内存优化

**创建日期**: 2026-06-25
**状态**: ✅ 已关闭（无需优化）
**优先级**: P1

## 问题描述

10000 文件列表占用约 336MB 内存，本项目用于转存超多文件，此内存占用可能导致 OOM。

## 根因分析

**结论：336MB 是 RSS 测量误导，实际 Python 对象只有 5.1MB，无需优化。**

详细分析：
- RSS (Resident Set Size) 包含 Python 内存分配器开销
- Python 的 pymalloc 会预分配大块内存但不释放回 OS
- 实际 Python 对象内存通过 tracemalloc 测量：**5.1MB**

## 验证数据

```
测试场景：10000 个文件 dict（每个含 fs_id, server_filename, isdir, path, size）

tracemalloc 测量结果：
- Python 对象内存：5.1MB
- RSS：~336MB（含分配器开销，会复用）
```

## 结论

1. **无 OOM 风险**：实际内存占用仅 5.1MB
2. **无需优化**：dict 方案完全可用
3. **RSS 会复用**：Python 分配器预分配的内存会被后续对象复用

## 关键发现

- `resource.getrusage` 测量 RSS，包含 Python 分配器开销
- `tracemalloc` 测量实际 Python 对象分配
- 对于小对象（5 个字段），dict 比 `__slots__` 更省内存
- 10000 文件场景完全可用，无需优化
